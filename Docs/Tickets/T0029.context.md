- Offline-only implementation.
- Keep scope constrained to startup progress visibility and deferred startup work.
- Progress must reflect real stages, not a fake spinner-only experience.
- Prefer stage text + elapsed time over invented percentages.
- Do not introduce new startup hangs, deadlocks, or cross-thread UI access bugs.
- Defer only non-critical startup work; do not weaken correctness.
