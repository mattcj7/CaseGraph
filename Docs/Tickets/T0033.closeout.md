# Ticket Closeout: T0033 - Shell window controls and desktop polish

## What shipped
- Added visible minimize, maximize/restore, and close buttons to the main shell title bar.
- Added title-bar drag behavior, double-click maximize/restore, and drag-from-maximized restore handling so the shell behaves like a normal Windows desktop window.
- Reserved a dedicated fixed-width top-right control strip so minimize/maximize/restore/close stay visible and clickable at narrow widths and in maximized/restored states.
- Made the title/header region the flexible portion of the shell so text truncates/clips before the window controls can be pushed offscreen.
- Fixed the startup XAML parse failure by replacing the invalid `ColumnDefinition.Width` static numeric binding with a valid fixed grid width literal for the control strip.
- Kept the change isolated to `MainWindow` so startup diagnostics, readiness flow, and feature navigation behavior remain unchanged.
- Added focused regression tests for the shell window-state helper logic and the reserved control-strip width contract in the existing WPF-capable infrastructure test project.
