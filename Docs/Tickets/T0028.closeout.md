# Ticket Closeout: T0028 - Startup and page activation stabilization sweep

## What shipped
- Added explicit structured startup-stage logging in `App.xaml.cs` for host build, workspace migration, host start, and main-window resolution/show so post-migration failures no longer look like silent hangs.
- Hardened `WorkspaceDbInitializer` after `ApplyMigrationsCompleted` by logging the `EnsureMessageFtsObjects` boundary, logging FTS ensure/backfill outcomes, and skipping the expensive backfill when the FTS table and triggers are already in sync.
- Fixed `ReportsViewModel` initialization order by assigning default observable state through the backing field during construction and by guarding side-effect handlers until initialization is complete.
- Added explicit Reports and Timeline case-context / activation lifecycle logs so tab activation failures surface with page-specific event names and exception context.
- Kept DI and navigation wiring intact for Reports, Timeline, and Incident Window; startup-critical registrations remain singleton-based and the `MainWindowViewModel` activation path still routes through the expected page/viewmodel boundaries.
- Added deterministic regression coverage for startup progression past migration completion, Reports and Timeline constructor/DI resolution safety, Reports activation after case selection, Timeline activation without a current case, and Reports/Timeline route creation.
