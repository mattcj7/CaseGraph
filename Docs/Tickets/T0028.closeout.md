# Ticket Closeout: T0028 - Reports startup null-safety hotfix

## What shipped
- Hardened `ReportsViewModel` constructor-time initialization so the default subject-kind state is assigned via the backing field before callback logic is allowed to run, preventing startup-time null/reference ordering faults.
- Added explicit startup-stage diagnostics in `App.xaml.cs` for host build, workspace migration completion, host start, and main-window resolution/show so post-migration stalls are identifiable from the log instead of presenting as a silent stop.
- Expanded `WorkspaceDbInitializer` diagnostics around the post-migration FTS reconciliation step and skipped the expensive FTS backfill when the table/triggers are already in sync, allowing startup to progress past `ApplyMigrationsCompleted` on steady-state workspaces.
- Added explicit Reports/Timeline activation and case-context failure logging so tab activation faults surface as named lifecycle events rather than only generic fire-and-forget failures.
- Added deterministic regression coverage for self-test startup progression past migration completion, Reports constructor/activation safety, Timeline constructor/activation safety, and Reports/Timeline navigation view creation.
