# Ticket: T0044 - Gang documentation packet export v1

## Goal
Add a template-driven Gang Documentation packet export so approved gang documentation records can be turned into a readable, defensible packet with citations, approval metadata, and linked supporting information.

## Context
CaseGraph now has:
- gang documentation child records under target/person profiles
- criteria/basis entries
- linked organizations and subgroup/set context
- supervisor approval workflow

The next logical step is a formal export packet for approved gang documentation. The packet should remain clearly distinct from the general target profile and should assemble documentation content, review metadata, and supporting linked information into a clean exportable package.

This export must be template-driven, citation-backed, auditable, and investigator/supervisor friendly.

## Scope
### In scope
- Add a Gang Documentation packet export workflow for approved gang documentation records.
- Add an export preview step before writing output.
- Export packet should include at minimum:
  - subject identity summary
  - linked organization
  - linked subgroup/set/clique if applicable
  - current gang documentation status
  - approval/reviewer metadata
  - review notes / decision notes where applicable
  - criteria checklist with basis summaries
  - linked supporting evidence references/citations
  - linked cases and incidents where already available
- Add clear labeling that separates:
  - evidence-derived content
  - analyst-entered basis/intel
  - workflow/reviewer metadata
- Preserve direct citations/provenance in the packet:
  - SourceEvidenceItemId and/or SourceLocator-backed references where applicable
- Add export audit logging.
- Keep packet assembly query-driven and based on existing indexes/services rather than denormalizing large evidence payloads into the documentation record.
- Add deterministic tests for export assembly and citation behavior.

### Out of scope
- Full DOCX/PDF styling perfection beyond a clean v1 export
- Batch export of many documentation records at once
- Organization hierarchy/pyramid visualization
- Rivalry/territory modeling
- Advanced narrative builder integration
- New review workflow features beyond what already exists

## Provenance & Audit Requirements
### Provenance
- Exported gang documentation packets must preserve citations back to underlying evidence and/or SourceLocator-backed supporting items.
- Packet content must clearly distinguish:
  - evidence-derived items
  - analyst-entered basis
  - supervisor/reviewer workflow metadata
- No export should weaken explainability or source traceability.

### Audit
- Log the export action with:
  - exported documentation record id
  - operator/user
  - timestamp
  - output path or export target summary
- Keep behavior consistent with existing CaseGraph export audit patterns.

## Performance & Resource Management Requirements
- Packet assembly must be query-driven and bounded.
- Do not load unnecessary large evidence payloads into memory.
- Export preview must remain responsive.
- Use safe async patterns and deterministic disposal for writers/streams.
- No UI blocking for long export operations.

## Implementation Notes / Edge Cases
- Only approved documentation should be exportable by default in v1.
- If a record is not approved, the UI should explain why export is unavailable.
- Preview should show what will be exported before writing output.
- Keep v1 practical and readable; do not overcomplicate formatting.
- The packet should pull from existing linked data where available:
  - organizations/sets
  - cases/incidents
  - supporting basis/citations
- This ticket is about packet export, not redesigning the documentation workflow.
