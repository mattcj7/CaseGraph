# Ticket: T0036 - Synthetic evidence generator window

## Goal
Create a small standalone synthetic evidence generator tool with a simple window that allows the user to generate multiple synthetic Cellebrite-like export packs for CaseGraph testing.

## Context
CaseGraph now has enough Timeline, Search, Association Graph, Incident Window, and Locations functionality that one-off manual test files are no longer enough. We need a repeatable way to generate realistic synthetic evidence datasets with controlled randomness so we can repeatedly import them into CaseGraph and test graphing, timeline reconstruction, location convergence, and search behavior.

A generator window is preferable to a single static pack because it allows:
- reproducible runs via seed
- multiple datasets per run
- adjustable scenario size/complexity
- ongoing QA as CaseGraph evolves

## Scope
### In scope
- Add a small standalone generator tool/window (not a new investigative feature inside the main CaseGraph workflow).
- The tool should let the user choose:
  - seed
  - dataset count
  - number of persons
  - approximate message count
  - approximate location count
  - scenario/profile
  - output folder
- Initial supported scenario/profile(s):
  - `offense_window` (required)
- Generate synthetic Cellebrite-like export packs with files such as:
  - `contacts.csv`
  - `message_threads.csv`
  - `messages.csv`
  - `locations.csv`
  - `locations.json`
  - `device_locations.plist`
  - `expected_findings.md`
  - `manifest.json`
  - `README.md`
- Output each generated dataset into its own folder.
- Keep data fictional and clearly synthetic.
- Generated datasets should be:
  - reproducible by seed
  - believable enough to exercise Timeline, Association Graph, Search, Incident Window, and Locations
- Add a minimal expected-findings guide per dataset:
  - suggested offense window
  - central subjects
  - likely strong graph links
  - expected location convergence
  - suggested QA searches
- Add basic validation and success/failure feedback in the tool window.

### Out of scope
- Real UFDR/UFDX generation
- Proprietary Cellebrite binary/container formats
- Large scenario library in v1
- Integration into the main CaseGraph investigation workflow
- Advanced styling/polish beyond a functional utility window

## Performance & Resource Management Requirements
- Generator should remain responsive while generating multiple datasets.
- Use bounded generation; avoid runaway outputs.
- Validate user inputs and guard against nonsensical sizes.
- Dispose file resources deterministically.

## Implementation Notes / Edge Cases
- Keep v1 focused and practical.
- The required profile is:
  - `offense_window`
- The offense window profile should include:
  - pre-event chatter
  - offense-window coordination
  - aftermath chatter
  - multi-party threads
  - location convergence near a scene
  - central and peripheral subjects
- Each dataset should produce:
  - a slightly varied but reproducible scenario
  - an `expected_findings.md` that explains what QA should see
- Manifest should include:
  - seed
  - profile
  - subject count
  - message count
  - location count
  - generated folder name
- Prefer a separate utility project/window over adding this into the main CaseGraph investigation shell.
- Keep the UI simple:
  - text boxes / numeric inputs
  - profile dropdown
  - output folder picker
  - generate button
  - status area

## How to Verify Manually
1. Launch the synthetic evidence generator tool.
2. Choose a seed, dataset count, person count, message count, location count, profile, and output folder.
3. Generate one dataset and inspect the folder contents.
4. Generate again with the same seed/settings and confirm reproducible output structure.
5. Import one generated dataset into CaseGraph and use it to test Timeline, Search, Association Graph, Incident Window, and Locations.
