# Ticket Closeout: T0037 - Message parser adaptability and canonical schema mapping

## What shipped
- Added a canonical internal message normalization contract in `CaseGraph.Infrastructure.Import` with explicit provenance, parser family/version, attachment metadata, and per-row parse warnings.
- Reworked `MessageIngestService` so CSV, XLSX, and UFDR rows normalize through the same canonical message path before persistence.
- Added header alias mapping and flexible CSV/XLSX detection based on canonical field matches plus sheet-name heuristics, rather than only exact template names.
- Added structured parser diagnostics for parser selection, canonical field matches, unmapped headers, and parsed/skipped/warning row counts.
- Verified that a T0036-style `messages.csv` export shape now ingests through the normal message ingest path.
- Added deterministic regression coverage for alias mapping, canonical normalization, partial-field warnings, CSV diagnostics, synthetic-export compatibility, and stabilized a flaky workspace-initializer timing test encountered during full-suite verification.
