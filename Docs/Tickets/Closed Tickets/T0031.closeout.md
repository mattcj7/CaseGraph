# Ticket Closeout: T0031 - Search readiness no-blocking and FTS maintenance manager

## What shipped
- Split Search readiness into a fast message-FTS status check plus deferred shared maintenance.
- Prevented the first keyword query from blocking behind heavy FTS repair/backfill work.
- Added a visible Search preparing state while maintenance completes.
- Delayed query execution until readiness finishes without freezing the app.
- Separated readiness/maintenance logging from actual search-query logging.
- Added deterministic regression coverage for Search readiness behavior.
