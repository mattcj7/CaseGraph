# Ticket: T0025 - Reports v1 - Target/Global Person Dossier Export (Citations)

## Goal
Add a **Reports** page and an export workflow to generate a **court-ready dossier** for a selected Target/Global Person, including message/timeline summaries and **evidence citations** (EvidenceItem + SourceLocator) for every claim shown.

## Context
We have Messages ingest + search, Targets + Global Persons, provenance drill-down, audit logging, and Timeline v1. A dossier export is a core deliverable for investigations and helps validate the platform’s defensibility.

## Scope
### In scope
- Add **Reports** page in left-nav.
- Add “Create Dossier” guided flow:
  - Select Target or Global Person
  - Choose date range (optional)
  - Choose included sections (checkboxes)
  - Choose output folder + filename
- Dossier content (v1):
  - Cover page: case name/id, subject name, generated timestamp, operator
  - Subject identifiers (phones/emails/handles) with “why linked” (manual link decision ids where available)
  - Where-seen summary (messages): totals, first/last seen, top counterparties (if already available)
  - Timeline excerpt (messages-only for now) within chosen range
  - Notable message excerpts: top N by recency (bounded) with citations
  - Appendix: citation list (EvidenceItem + SourceLocator)
- Output format:
  - **PDF** (preferred) OR HTML (print-ready) if PDF is too heavy for v1
- Audit:
  - Log `ReportExported` with parameters (subject id, date range, sections, output path)

### Out of scope
- Calls/locations/media sections (future)
- Auto-generated narrative “analysis” text (future / optional AI assist later)
- Bulk exports for multiple subjects
- Fancy styling/themes (keep clean and readable)

## Acceptance Criteria (Testable)
- [ ] Reports page exists and provides a “Create Dossier” workflow.
- [ ] User can export a dossier for a Target/Global Person to a chosen folder.
- [ ] Every section that references evidence includes citations:
  - [ ] EvidenceItem id
  - [ ] SourceLocator
- [ ] Excerpts/snippets are bounded (no massive message bodies).
- [ ] Export does not freeze UI; runs via job queue or async background with progress + cancellation.
- [ ] Export action is audited as `ReportExported` with key parameters.
- [ ] Deterministic tests cover dossier data selection rules and audit logging.

- None expected.

## Provenance & Audit Requirements
### Provenance
- All evidence-derived content included must provide:
  - SourceEvidenceItemId
  - SourceLocator
- If a section is purely computed (counts), it must still cite the underlying evidence set (e.g., list of MessageEventIds or representative citations).

### Audit
Log:
- `ReportExported` with:
  - CaseId
  - SubjectType + SubjectId
  - DateRange (if any)
  - IncludedSections
  - OutputPath (and/or file hash if already supported)
  - CorrelationId

## Performance & Resource Management Requirements
- Run export off the UI thread (JobQueue or async background).
- Support CancellationToken.
- Stream writing output; do not hold huge strings/blobs in memory.
- Dispose all streams deterministically.

## Implementation Notes / Edge Cases
- If subject has no messages, export still succeeds with clear “no data” sections.
- File path:
  - sanitize filename
  - avoid overwriting unless user confirms (or auto-suffix)
- Citations must be copyable and consistent with existing “Copy citation” formatting.
- Keep styling simple and readable (court-friendly).

## Test Plan
### Unit tests
- Section toggles include/exclude correct content.
- Date range filters restrict timeline/excerpts.
- Every excerpt entry contains at least one citation.

### Integration tests (if applicable)
- Seed minimal workspace with messages + target linking.
- Export dossier.
- Verify:
  - file created
  - audit entry written
  - output contains subject + at least one citation

## How to Verify Manually
1. Open a case with ingested messages and a target/global person.
2. Go to Reports → Create Dossier.
3. Pick subject + date range + sections.
4. Export to a folder.
5. Open the output and confirm citations exist and “no data” sections behave.
6. Confirm audit log shows `ReportExported`.
