# Ticket Closeout: T0029 - Startup progress window and deferred startup work

## What shipped
- Added a startup progress window that appears before host build completes, shows real startup stage text from the actual startup/workspace-init pipeline, and updates elapsed time while startup is running.
- Wired startup-stage reporting through a shared Core diagnostics contract so `App.xaml.cs` and `WorkspaceDbInitializer` report the same live stage stream without breaking project boundaries or existing structured startup logging.
- Preserved startup failure visibility by surfacing failure state and diagnostics path in the progress window before the existing fatal-error flow shuts the app down.
- Deferred non-critical stale-running-job cleanup until after the main window is shown, while keeping critical workspace open/migration/schema verification work on the blocking startup path.
- Added deterministic unit coverage for startup stage reporting and progress viewmodel behavior.
