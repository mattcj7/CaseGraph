# Ticket: T0042 - ZIP evidence ingest and archive detector hardening

## Goal
Make ZIP archives a first-class evidence ingest path so investigators can import a single archive instead of manually uploading hundreds of files, while preserving immutability, controlled extraction, parser routing, and precise provenance to inner files.

## Context
Many real returns will arrive as ZIP archives, especially social media warrant returns, HTML exports, and bundled artifact sets. CaseGraph already expects ZIP-HTML support architecturally, but the ingest path needs to be hardened so ZIP imports are practical, trustworthy, and easy to use.

The desired workflow is:
- import the ZIP as one immutable evidence item
- hash and preserve the original archive
- extract to a controlled derived folder
- detect/archive-route inner contents safely
- parse supported artifacts from the extracted structure
- preserve provenance back to both the ZIP evidence item and the inner file/element/row path

This ticket is about ZIP ingest foundation and archive detector hardening, not every possible platform-specific parser edge case.

## Scope
### In scope
- Treat `.zip` archives as first-class EvidenceItems in the evidence vault.
- Preserve the original ZIP immutably with hash/manifest handling consistent with existing evidence rules.
- Add controlled extraction for ZIP evidence into a derived/extracted folder under workspace-managed rules.
- Add archive inspection and detector routing over extracted contents.
- Support ZIP-based parsing flow for at least:
  - zipped HTML / social media return style structures
  - extracted CSV/JSON/HTML/plist inner files where existing detectors/parsers already apply
- Add parser routing from extracted contents without requiring the user to manually unpack archives.
- Preserve provenance so derived records can trace back to:
  - the original ZIP EvidenceItem
  - extracted relative path
  - inner file row / DOM locator / key-path where applicable
- Add fail-soft behavior:
  - one bad inner file does not fail the whole archive ingest job
  - warnings/skips are logged clearly
- Add structured logging for:
  - archive import start/completion
  - extraction start/completion
  - detector matches
  - parser runs from archive contents
  - skipped/failed inner files
- Add fixture/regression coverage using ZIP test bundles.

### Out of scope
- Support for every proprietary archive/container format
- Full parser diagnostics UI
- Deep platform-specific social parser expansion beyond what existing detectors/parsers can already support cleanly
- Password-protected ZIP support unless trivially already available
- Broad redesign of the evidence import UI

## Provenance & Audit Requirements
### Provenance
- Every derived record created from archive contents must preserve:
  - `SourceEvidenceItemId` pointing to the original ZIP evidence item
  - `SourceLocator` that identifies the inner file and inner row/DOM/key-path when applicable
  - `IngestModuleVersion`
- ZIP extraction must not weaken evidence traceability.

### Audit
- Add structured logs and/or audit entries for:
  - archive imported
  - extraction completed
  - archive parse run
  - warnings/skips/failures
- Keep behavior consistent with existing CaseGraph audit and diagnostics patterns.

## Performance & Resource Management Requirements
- ZIP extraction and archive parsing must not block the UI thread.
- Archive work must run in background/job flows with progress where applicable.
- Dispose streams, archives, and temp/extraction resources deterministically.
- Avoid loading the full archive into memory when not needed.
- One malformed inner artifact must not fail the overall archive ingest.

## Implementation Notes / Edge Cases
- Prefer a controlled extracted folder pattern rather than ad hoc temp extraction.
- Preserve relative paths inside the archive for provenance and debugging.
- Detector routing should inspect extracted contents and apply existing parser families where possible.
- HTML parsing should continue to use DOM parsing, not regex.
- Keep v1 practical:
  - ZIP import
  - extraction
  - detector routing
  - fail-soft parsing
  - provenance
- Do not try to solve every archive family in one pass.

## How to Verify Manually
1. Import a ZIP archive containing supported artifact files.
2. Confirm the ZIP appears as a single evidence item in the vault.
3. Run ingest and confirm extraction occurs automatically.
4. Confirm supported inner files parse without manual unpacking.
5. Confirm unsupported or malformed inner files do not crash the ingest job.
6. Open parsed results and confirm provenance points back to the original ZIP and inner file locator.
