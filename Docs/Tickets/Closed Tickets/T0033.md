# Ticket: T0033 - Shell window controls and desktop polish

## Goal
Make CaseGraph behave like a polished Windows desktop application by implementing standard shell window controls and basic desktop window interactions.

## Context
Recent reinforcement work improved startup, readiness behavior, and performance visibility. The next usability gap is that the app shell still lacks standard desktop window behavior such as minimize, maximize/restore, and close. These are baseline expectations for trust and usability in a Windows desktop application.

## Scope
### In scope
- Implement working shell window controls:
  - minimize
  - maximize / restore
  - close
- Implement expected desktop interactions:
  - drag window by title bar
  - double-click title bar to maximize / restore
  - proper restore from maximized state
- Ensure shell state stays visually correct when switching between normal/maximized states.
- Ensure controls work consistently with the existing app shell and startup flow.
- Add any required viewmodel/command wiring for shell window actions.
- Preserve existing startup diagnostics and error handling behavior.

### Out of scope
- Full shell visual redesign
- Theme overhaul
- Multi-window support
- System tray support
- Reworking unrelated app navigation or feature flows

## Acceptance Criteria (Testable)
- [ ] Minimize button minimizes the window.
- [ ] Maximize button maximizes the window.
- [ ] Restore works correctly from maximized state.
- [ ] Close button closes the app normally.
- [ ] User can drag the window by the title bar.
- [ ] Double-clicking the title bar toggles maximize/restore.
- [ ] Window chrome behavior remains stable after startup and during case/feature navigation.
- [ ] `dotnet test` passes.

## Implementation Notes / Edge Cases
- Prefer standard Windows behavior over custom/non-standard interactions.
- If current shell uses custom chrome, implement controls carefully so resize/maximize/restore behavior remains correct.
- If standard chrome can be restored with minimal disruption and better reliability, that is acceptable if it stays within scope and improves usability.
- Keep this ticket focused on shell/window behavior only.

## How to Verify Manually
1. Launch the app.
2. Click minimize and confirm the app minimizes.
3. Restore the app and click maximize.
4. Confirm maximize/restore toggles correctly.
5. Double-click the title bar and confirm maximize/restore works.
6. Drag the title bar and confirm the window moves normally.
7. Click close and confirm app exits normally.
