# Ticket Closeout: T0042 - ZIP evidence ingest and archive detector hardening

## What shipped
- Added ZIP archive handling to the evidence vault so `.zip` imports stay immutable in `vault/<evidenceId>/original/...` and are extracted into `vault/<evidenceId>/derived/extracted/...` with archive extraction manifests and structured extraction logs.
- Routed ZIP inner files through existing ingest services:
  - message ingest now inspects extracted `.csv`, `.xlsx`, `.ufdr`, and `.html` entries
  - locations ingest now inspects extracted `.csv`, `.json`, and `.plist` entries
- Preserved archive provenance by keeping the original ZIP `EvidenceItemId` on derived rows and writing inner-file locators that point back to the extracted archive-relative path.
- Added fail-soft archive handling so malformed inner files are logged and skipped without aborting the overall archive ingest run.
- Updated the import picker/import page text so ZIP archives are presented as a first-class evidence path.
- Added deterministic regression coverage for ZIP import/extraction, archive parser routing, fail-soft inner-file handling, and provenance.
