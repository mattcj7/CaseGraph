# T0041 Closeout - Gang documentation module v1

## What shipped
- Added a first-class Gang Documentation persistence model with record, criterion, and workflow-history tables plus the `GangDocumentationV1` migration.
- Added `IGangDocumentationService` and `GangDocumentationService` with create/update/load flows, criteria save/remove flows, organization and subgroup linkage validation, and append-only audit/diagnostic events.
- Added a dedicated Gang Documentation section under person/target profiles with separated inline editing for formal documentation, workflow/reviewer/review-due fields, criteria checklist management, and workflow history display.
- Reused existing target/global-person and organization foundations by linking documentation records to `TargetRecord`, optional `PersonEntity`, and existing organization/subgroup records.
- Added deterministic infrastructure tests for documentation persistence, criteria persistence, organization/subgroup linkage, and workflow/review updates, and extended the profile viewmodel test to load the new module.
- Hotfixed the profile UI workflow so the empty-state now initializes a real create-record flow, defaults the editor to a usable organization-backed state, disables criteria actions until a record is actually saved, and exposes obvious next-step guidance instead of a dead-end panel.
- Added regression coverage for the Gang Documentation viewmodel create/save/reload path and the criterion add/save/reload path.
- Hotfixed organization/subgroup linkage visibility so saved gang documentation now clearly shows the linked organization and linked subgroup/set in the profile list and editor state after save/reload.
- Added submit-attempt validation UX for required gang documentation and criterion fields, with red asterisks that stay visible until corrected, inline validation guidance, and focus handoff to the first invalid field where practical.
- Extended Organization profile roster review so each member row now surfaces gang documentation state (`Documented`, `Pending Review`, `Draft`, `Inactive`, or `No Documentation`) and, when present, a compact organization/subgroup linkage summary.
- Added member-row quick actions that route investigators to the person profile with action labels that distinguish `Open Documentation` from `Create Documentation`.

## Manual verification
1. Run `dotnet test 'CaseGraph/CaseGraph.sln' --no-restore`.
2. Open a person/target profile in the app.
3. In the Gang Documentation section, create a record linked to an organization and optional subgroup/set.
4. Add criteria entries with basis summaries, observed dates, and source notes.
5. Update workflow status, approval status, reviewer, and review-due fields, then save.
6. Refresh the profile and confirm the documentation, linked organization/subgroup, criteria, and workflow history reload under the dedicated Gang Documentation section.
7. Attempt to save with required fields blank and confirm the missing fields show red asterisks until corrected.
8. Open an Organization profile and confirm each member row shows either a gang documentation state or `No Documentation`.
