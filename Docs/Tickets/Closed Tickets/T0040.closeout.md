# T0040 Closeout - Entity Profile Viewer Navigation for People and Organizations

## Summary
Implemented dedicated profile viewer navigation for People / Targets and Organizations.

This ticket focused on making entity records feel like first-class navigable case objects by adding drill-in profile pages, affiliation visibility in People / Targets results, and persistent labels/empty states on affected screens.

## What was completed
- Added dedicated Person / Target Profile page
- Added dedicated Organization Profile page
- Added double-click navigation from list results
- Added navigation from organization memberships to person profiles
- Added navigation from person affiliations to organization profiles
- Added affiliation summary display in People / Targets results
- Replaced hover-only field discovery with visible labels/section headers where required
- Added empty-state messaging for missing data sections
- Preserved existing create/save/edit behavior
- Preserved existing audit behavior for edit/link operations without adding passive navigation audit noise

## Notes / edge cases encountered
- Membership drill-in resolves the current-case target from `GlobalEntityId`; if none exists, the person profile shows a non-crashing unavailable state.
- Profile navigation uses an in-memory workspace history stack only; it does not create audit entries.
- Affiliation summaries are derived from existing organization membership rows and stay read-only/query-driven.
