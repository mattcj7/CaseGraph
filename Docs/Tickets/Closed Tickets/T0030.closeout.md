# Ticket Closeout: T0030 - Case-open lazy initialization and feature-scoped readiness

## What shipped
- Split readiness into explicit startup, case-open, and feature-open phases.
- Removed message-search FTS reconciliation from global startup so the shell only waits for host build, migration safety checks, and schema verification.
- Added `CaseOpenReadinessService` so case open visibly runs bounded readiness work before the selected case is published across the UI.
- Added `FeatureReadinessService` so Search and Timeline request message-search readiness only when they actually need keyword-index work, while Reports and Incident Window perform explicit no-op readiness checks with phase-labeled diagnostics.
- Changed Timeline, Reports, Incident Window, and Locations to stop eagerly loading feature data on every case switch when those pages are inactive.
- Preserved existing diagnostics and added phase-labeled readiness logs for startup, case open, and feature open.
- Added deterministic regression coverage for the startup/readiness split plus case-open/feature readiness orchestration.
