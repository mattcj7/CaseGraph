# Ticket: T0039 - Gang organization registry and membership v1

## Goal
Create a first-class gang organization registry so CaseGraph can track gangs, sets, cliques, crews, and subgroup structures across cases, and link global people to those organizations with roles, status, confidence, and basis information.

## Context
CaseGraph now has core investigation workflows including search, graphing, timeline support, incident workspace, and cross-case identity concepts. The next logical foundation is a gang intelligence layer that is not tied to a single case. A gang, set, clique, or crew may appear across many investigations, and a person may belong to or associate with multiple organizations over time.

This ticket should create the durable organization and membership model needed for:
- gang files
- subgroup/set tracking
- cross-case organization intelligence
- future hierarchy/pyramid visualization
- incident-to-organization linkage
- gang documentation/validation workflows

## Scope
### In scope
- Add a first-class **Organization Registry** with support for organization types such as:
  - gang
  - set
  - clique
  - subgroup
  - crew
  - hybrid / informal group
- Add organization records with at least:
  - organization id
  - name
  - type
  - status (active/inactive)
  - parent organization id (optional)
  - notes/summary
- Add organization aliases.
- Add organization membership records linking a global person to an organization.
- Membership records should support at least:
  - membership id
  - global person id
  - organization id
  - role
  - status (member / associate / former / suspected / affiliate)
  - confidence
  - basis summary
  - start date
  - end date
  - last confirmed date
  - reviewer / review notes if lightweight in v1
- Add UI flows to:
  - create an organization
  - edit an organization
  - add aliases
  - add/remove/update memberships for global people
  - browse organizations and members
- Add support for linking a case to one or more organizations with simple relevance notes/confidence if feasible in v1.
- Preserve audit/provenance patterns where applicable.
- Add structured diagnostics/logging for organization create/update/link flows.

### Out of scope
- Full organization hierarchy/pyramid visualization
- Rivalry modeling
- Territory modeling
- Validation packet/report generation
- Incident-to-organization scoring logic
- Advanced review workflows beyond lightweight fields
- Broad intelligence dashboarding

## Provenance & Audit Requirements
### Provenance
- Organization membership basis should support evidence-backed notes or summary references without weakening provenance elsewhere.
- Existing global person and cross-case identity traceability must remain intact.

### Audit
- Add structured logs or audit entries for:
  - organization created
  - organization updated
  - alias added/removed
  - membership created/updated/removed
- Keep behavior consistent with existing CaseGraph audit/diagnostic patterns.

## Performance & Resource Management Requirements
- Organization registry queries should be responsive.
- Avoid heavy cross-case scans on every page load.
- Keep browsing/editing bounded and indexed where appropriate.
- Use SQLite-safe query patterns.
- Dispose resources deterministically.

## Implementation Notes / Edge Cases
- Keep v1 focused on durable organization records and memberships, not yet on advanced gang intelligence workflows.
- Parent organization should allow:
  - gang -> set
  - gang -> clique
  - subgroup -> crew
- Membership status should avoid overstating certainty:
  - member
  - associate
  - former
  - suspected
  - affiliate
- Time-bounded fields matter:
  - start date
  - end date
  - last confirmed date
- Basis summary can be lightweight in v1, but should be durable enough to support later validation workflows.
- Reuse the existing global person model instead of creating a duplicate person identity system.
- Keep this ticket foundational and data-model centered.
