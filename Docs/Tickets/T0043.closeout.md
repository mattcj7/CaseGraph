# Ticket Closeout: T0043 - Gang documentation supervisor approval workflow v1 hotfix

## What shipped
- Kept the formal gang documentation workflow states for Draft, Pending Review, Approved, Returned for Changes, Inactive, Purge Review, and Purged records, but made the purge side reversible instead of a one-way dead end.
- Updated the workflow transition service so only valid next actions are allowed from the current state, including:
  - `Returned for Changes -> Save as Draft` through content edits
  - `Inactive -> Restore to Approved`
  - `Purge Review -> Restore to Approved | Restore to Inactive | Purge`
  - `Purged -> Restore to Approved | Restore to Inactive`
- Preserved dedicated review/history persistence with reviewer attribution, decision notes, review timestamps, and durable status history entries for forward and restore transitions.
- Updated the Gang Documentation UI to show a clear workflow snapshot with current status, next allowed actions, reviewer/timestamp summary, latest decision note, and only the action buttons valid for the selected state.
- Kept gang documentation content and criteria editable only in Draft or Returned for Changes, with Returned for Changes edits explicitly saving the record back to Draft before re-submission.
- Added deterministic regression coverage for valid forward transitions, reversible purge-review transitions, restoring a purged record, invalid transition blocking, and UI clarity for Returned for Changes and Purged states.
