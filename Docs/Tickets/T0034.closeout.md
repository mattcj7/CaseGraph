# Ticket Closeout: T0034 - Background maintenance manager and readiness banners

## What shipped
- Added a reusable `IBackgroundMaintenanceManager`/`BackgroundMaintenanceManager` with shared task registration, lifecycle snapshots, duplicate-request deduplication, optional cancellation, and structured queued/started/completed/failed/deduplicated logs.
- Routed deferred message-search maintenance through the shared manager in `FeatureReadinessService` so Search and Timeline can attach to the same in-flight background task instead of scheduling overlapping work.
- Added a reusable `ReadinessBanner` control plus shared banner-state models/mapping for Pending/Running/Completed/Failed maintenance visibility.
- Surfaced maintenance banners in Search, Timeline, Reports, and Incident Window, while preserving existing readiness status text, provenance actions, diagnostics, and performance instrumentation.
- Kept Search and Timeline responsive during deferred maintenance by continuing the original request after shared background maintenance completes instead of blocking the UI thread.
- Added deterministic regression coverage for manager lifecycle, deduplication, failure state, banner-state mapping, and Timeline banner subscription behavior.

## Files changed (high-level)
- App services: background maintenance contracts/models/manager, feature-readiness integration, DI registration.
- App viewmodels/views: Search/Timeline/Reports/Incident Window maintenance banner state and UI.
- Tests: maintenance manager lifecycle/dedupe/banner regressions plus readiness/viewmodel constructor updates.
- Docs: context whitelist correction, this closeout, completed ticket index entry.
