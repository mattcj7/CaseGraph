# Ticket Closeout: T0038 - Open incident workspace v1

## What shipped
- Added persisted Open Incident Workspace v1 data model and migration for incidents, scene locations, and pinned findings.
- Added `IIncidentService`/`IncidentService` for incident create-update persistence, offense-window and multi-scene radius cross-reference, incident timeline anchor generation, and durable pinning.
- Hotfixed incident refresh/load for SQLite by materializing filtered incident rows before applying in-memory `UpdatedUtc` and `Title` ordering, preserving incident list order without unsupported `DateTimeOffset` SQL `ORDER BY` translation.
- Added a first-class WPF Open Incident Workspace page/viewmodel on the existing incident navigation slot with incident list/editor, message and location result views, timeline markers, and pinned findings.
- Preserved provenance surfaces by carrying source evidence ids, locators, citations, and View Source / Copy Citation actions through incident results and pinned findings.
- Added deterministic SQLite-backed infrastructure tests covering incident persistence, incident ordering, offense-window filtering, radius filtering, timeline anchor generation, and pin persistence.

## Files changed (high-level)
- Infrastructure: incident domain/service layer, new persistence entities, workspace `DbContext`, migration, and initializer required-table list.
- App: DI registration, navigation wiring, main window incident-workspace integration, and new Open Incident Workspace view/viewmodel.
- Tests/docs: new incident service coverage, this closeout, and completed-ticket index entry.
